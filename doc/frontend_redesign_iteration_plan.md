# KeyRecord 前端显示重构与设计迭代计划

## 1. 文档信息

- 文档状态：待实施
- 目标页面：KeyRecord 本地键盘统计可视化页面
- 实施范围：`visualize/public/` 静态前端
- 桌面验收基准：1440px 宽视口
- 参考设计图：[KeyRecord 仪表盘设计参考](./keyrecord_dashboard_design_reference.png)
- 参考图来源：需求方提供的原始 PNG，仓库内文件为无损副本

> 参考图用于确定页面结构、颜色、密度、层级和交互入口。图中的日期、按键次数、比例等均为示例，不作为真实数据口径或测试期望值。

![KeyRecord 仪表盘设计参考](./keyrecord_dashboard_design_reference.png)

## 2. 背景与当前实现

当前可视化页面使用原生 HTML、CSS、JavaScript，图表基于 D3 和 heatmap.js。静态资源会在 CMake 构建阶段嵌入 C++ 可视化服务，不存在独立的前端构建工具链。

现有页面已经具备以下能力：

- 数据库总览与可用日期范围；
- 手工日期范围、今天、最近 7 天、最近 30 天、本月、全部等筛选；
- 日期 Brush、每日统计日历和单日点击联动；
- 键盘热力图、缩放、PNG 导出和按键悬浮详情；
- Top 20 按键排行以及 CSV、JSON 导出；
- 小时热力图、输入速度、按键分区、左右手、组合键、时序回放；
- 实时监控、范围对比、深浅主题和打印。

本次工作不是重新选择技术栈，而是在保留现有数据能力的基础上重组页面结构、统一视觉系统，并让所有主要模块遵循同一日期范围和数据口径。

## 3. 已确认的产品决策

| 决策项 | 最终约定 |
| --- | --- |
| 页面形态 | 单页长页，不拆路由；侧栏使用锚点导航和滚动联动高亮 |
| 适配范围 | 仅将桌面端作为本轮正式验收范围 |
| 默认日期 | 以数据库最新日期为结束日，默认选择最近 30 个自然日，并受数据库首日限制 |
| 指标联动 | 顶部总览卡片和所有主要分析模块都跟随当前日期范围变化 |
| 日期主按钮 | 设计图中疑似“导入”的按钮按“应用”实现，不增加数据导入能力 |
| 键盘视图 | 在标准完整键盘和仅主键区的紧凑键盘之间切换 |
| 紧凑键盘范围 | 只保留字母、数字、标点、修饰键、空格等主键区；隐藏 F 区、导航区、方向键和数字小键盘 |
| 活跃周期口径 | 统计当前范围内次数大于零的活跃日期数量，不要求连续 |
| 旧功能 | 深色主题、打印、JSON 导出、键盘缩放、实时监控、时序回放和范围对比继续保留在所属区域 |
| 右上按钮 | 旧功能优先，两个按钮继续执行打印和主题切换 |
| 设置与帮助 | 左下入口只做视觉占位，禁用并提示“暂未开放” |
| 后端范围 | 不新增、不修改公共 API，不引入新的数据写入能力 |
| 依赖策略 | 不新增前端依赖；图标使用内联 SVG 或现有原生能力 |

### 4.1 页面结构

1. 固定侧栏与品牌区；
2. 顶部总览指标卡；
3. 日期范围与日活趋势；
4. 键盘热力图；
5. Top 20 按键排行；
6. 高级分析；
7. 每日按键统计。

侧栏六个主菜单分别指向对应区块。点击时平滑滚动，页面自然滚动时使用 `IntersectionObserver` 更新当前菜单。锚点定位需要为固定页面边距预留 `scroll-margin-top`，避免标题贴住视口顶部。

### 4.2 桌面尺寸基线

- 设计基准视口：1440px；
- 最小正式验收宽度：1280px；
- 补充验收宽度：1920px；
- 固定侧栏：224px；
- 主内容左右内边距：24px；
- 1440px 下主内容有效宽度约 1168px；
- 区块纵向间距：20px；
- 主卡片内边距：20px；
- 分析子卡片内边距：16px；
- 不新增移动端抽屉、底部导航或单独交互设计。

小于 1280px 时可以保留必要的横向滚动，但不允许在正式桌面尺寸出现内容重叠、表格列遮挡或键盘被裁切。

- 日期柱状图：主色 `#5268FF`，悬浮时提高饱和度；
- 键盘热力图：`#5A6FFF → #AEB4FF → #F3B6DC → #FF4F72`；

- 左右手环图：左手蓝色、右手紫色，双手和未知状态使用低饱和辅助色。

### 5.3 字体与组件

- 字体栈：`"Microsoft YaHei UI", "PingFang SC", "Noto Sans CJK SC", sans-serif`；
- 品牌名和主指标使用 600–700 字重，正文使用 400–500；
- 卡片圆角使用 10px 或 12px，不混用大量不同半径；
- 主阴影使用 `0 2px 10px rgba(20, 31, 60, 0.06)`；
- 普通按钮高度 36px，图标按钮 36×36px，日期输入高度 40px；
- 所有图标提供 `aria-label` 和 `title`，不依赖图形本身表达唯一语义；
- 深色主题继续使用语义变量映射，不要求与参考图进行像素级对照，但必须保证层级、对比度和图表可读性。

## 6. 数据口径与联动规则

### 6.1 全局日期状态

前端保留唯一的 `currentRange`：

```text
currentRange.start  YYYY-MM-DD
currentRange.end    YYYY-MM-DD
```

所有筛选入口最终都只更新该状态，再触发统一刷新流程。不得让键盘、排行、总览卡片各自维护独立的主日期范围。

日期校验规则：

- 起止值必须同时存在；
- 格式必须是合法的 `YYYY-MM-DD`；
- 起始日期不得晚于结束日期；
- 日期不得超出数据库 `first_date` 与 `last_date`；
- 校验失败时保留旧图表和旧范围，只在日期区显示错误；
- 最近 7 天、最近 30 天均包含结束日当天；
- “全部”使用数据库首日到末日；
- 空数据库时禁用范围按钮并显示无数据状态。

### 6.2 总览卡片

总览指标全部基于当前日期范围内的每日统计：

| 指标 | 计算方式 |
| --- | --- |
| 总按键数 | 当前范围每日 `count` 求和 |
| 活跃天数 | 当前范围内 `count > 0` 的日期数量 |
| 平均每日按键 | 总按键数 ÷ 活跃天数；活跃天数为零时显示 0 |
| 统计周期 | 起止日期及包含首尾的自然日数量 |

指标格式使用本地千分位；平均值按整数四舍五入；无数据时显示 `0` 和明确空状态，不使用短横线掩盖真实零值。

### 6.3 活跃概览

高级分析中的四项活跃指标定义为：

- 范围内活跃天数：与顶部活跃天数口径一致；
- 最活跃的一天：当前范围内 `count` 最大的日期、星期和次数；同值时选择较晚日期；
- 平均每小时输入：总次数 ÷ 当前范围自然日数 ÷ 24；
- 平均每分钟输入：总次数 ÷ 当前范围自然日数 ÷ 1440。

平均每小时保留整数，平均每分钟保留一位小数。当前范围无自然日或无数据时统一显示 0。

### 6.4 Top 20 占比

Top 20 每行占比统一为：

```text
按键当前范围次数 / 当前范围总按键数
```

不得继续使用“Top 20 榜内次数之和”作为分母。页面显示两位小数，进度条宽度使用相同未格式化比例计算。

## 7. 交互规格

### 7.1 侧栏

- 点击主菜单平滑滚动到目标区块；
- 自然滚动时高亮视口内占比最高或刚越过定位线的区块；
- 点击当前菜单仍执行定位，不触发重复数据请求；
- 设置、帮助按钮使用禁用态，悬浮或聚焦时显示“暂未开放”；
- 不为占位按钮注册空点击处理函数。

### 7.2 日期概览

- 手工修改日期后，只有点击“应用”或按 Enter 才提交；
- 快捷按钮直接提交并更新选中态；
- 默认选中“最近 30 天”；
- 柱状图展示当前日期范围，每个自然日占一个槽位，缺失日期补零；
- 悬浮柱形显示日期和次数；
- 点击单柱将全局范围切换为该日；
- Brush 在当前图表范围内拖选连续日期，结束时提交新范围；
- 图表底部展示当前范围总数和单日最高值；
- 日期输入、快捷按钮、柱形点击、Brush 必须走同一刷新入口。

### 7.3 键盘热力图

- 标准视图显示完整键盘；
- 紧凑视图只显示主键区；
- 视图切换复用当前热力数据，不重新请求 API；
- 切换后重建布局坐标、重算热力覆盖位置并重置缩放；
- 缩小、重置、放大继续使用现有 D3 Zoom；
- 刷新按钮清除热力图对应 API 缓存并只重新请求当前键盘数据；
- 下载按钮继续输出当前视图 PNG，文件名包含日期范围和布局模式；
- 键帽悬浮显示键名、VK Code、次数和当前范围占比；
- 高热值使用颜色强调，不通过发光动画制造干扰。

### 7.4 Top 20

- 桌面端左右分栏显示 1–10 和 11–20；
- 少于 20 条时按真实条数结束，不制造空排行；
- 表头、列宽和数字对齐保持两栏一致；
- 行悬浮仅使用浅主色背景；
- 模块内“导出 CSV”只导出当前范围 Top 20；
- 日期区现有全量 CSV、JSON 导出继续保留，其内容口径与当前范围一致。

### 7.5 高级分析

设计主网格为三列两行：

1. 小时热力图：当前范围的星期 × 小时统计；
2. 输入速度曲线：继续展示当前范围结束日的 5 分钟桶数据，并明确聚焦日期；
3. 按键分区统计：按次数降序展示，条形长度与比例一致；
4. 左右手使用对比：环图中心展示主要手别占比；
5. 快捷键统计：展示当前范围 Top 8 组合键；
6. 活跃概览：展示本文件定义的四项指标。

现有功能放置规则：

- 实时监控继续位于高级分析标题行；
- 时序回放保留在主网格之后，仍以当前范围结束日为回放日期；
- 范围对比保留在时序回放之后，A/B 范围独立于全局范围；
- 进入高级分析视口前不主动加载低频接口，继续使用按需加载；
- 实时刷新时先失效 API 缓存，再复用统一刷新流程，避免局部数据口径不一致。

### 7.6 每日按键统计

- 日历仅渲染当前日期范围，缺失日期以零值浅灰单元格显示；
- 单元格悬浮显示完整日期、星期和次数；
- 点击单日会把全局范围切换为该日并刷新全部模块；
- 图例保持五级，从“少”到“多”；
- 当前范围跨度较大时允许模块内部横向滚动，不压缩到不可读尺寸。

### 7.7 顶部操作与打印

- 右上打印按钮执行 `window.print()`；
- 右上主题按钮切换深浅主题并继续使用 `localStorage` 保存；
- 打印样式隐藏侧栏、日期交互工具、导出按钮、实时开关和回放控制；
- 打印内容保留标题、当前日期范围、主要图表和数据表格；
- 打印不得因固定侧栏留下 224px 空白。

## 8. 迭代阶段

每个阶段通过自身验收后再进入下一阶段，避免一次性修改全部 DOM、样式和图表逻辑后难以定位偏差。

### 阶段 1：页面骨架与设计令牌

实施内容：

- 重构页面为固定侧栏与主内容结构；
- 增加品牌区、六个锚点菜单和两个占位入口；
- 将颜色、圆角、阴影、间距、字号集中为 CSS 变量；
- 统一卡片、按钮、输入框、状态标签、表格和 Tooltip 基础样式；
- 接入滚动定位与菜单高亮；
- 调整打印布局以排除固定侧栏。

阶段验收：

- 1440px 下页面骨架与参考图比例一致；
- 侧栏滚动期间固定，主内容不被遮挡；
- 六个菜单均能准确定位；
- 深浅主题和打印仍可使用；
- 不修改任何统计接口和数据逻辑。

### 阶段 2：总览卡片与日期趋势

实施内容：

- 建立统一的每日统计缓存与范围聚合函数；
- 新增四张总览卡片；
- 将当前日期 Brush 重构为柱状趋势图并保留拖选；
- 统一手工日期、快捷范围、柱形点击和 Brush 的状态入口；
- 将默认范围从最近 7 天调整为最近 30 天；
- 增加总数、最高值、加载、空数据和错误状态。

阶段验收：

- 四项指标与选中范围严格一致；
- 缺失日期补零且不会改变活跃天数；
- 所有筛选入口产生相同的下游刷新结果；
- 快速连续切换范围时最终显示最后一次选择的数据。

### 阶段 3：键盘与排行

实施内容：

- 按参考图重做键盘容器、键帽和热力图色阶；
- 为键盘布局数据增加区域标记；
- 实现标准/紧凑布局切换；
- 整合刷新、PNG 下载和现有缩放操作；
- 将 Top 20 改为两栏连续排行；
- 修正占比分母，并增加模块内 CSV 导出。

阶段验收：

- 标准布局所有现有可映射键位置正确；
- 紧凑布局不显示 F 区、导航区、方向键和数字小键盘；
- 切换布局后热力覆盖、Tooltip 和 PNG 坐标正确；
- Top 20 排名、占比、进度条和 CSV 内容一致。

### 阶段 4：高级分析与每日统计

实施内容：

- 按三列两行建立设计主网格；
- 统一小时热力图、折线图、条形图、环图和统计卡视觉；
- 新增活跃概览聚合与渲染；
- 将时序回放、范围对比保留在扩展区域；
- 调整每日统计日历的范围、色阶、图例和提示；
- 保持高级分析按需加载和实时刷新。

阶段验收：

- 六张主卡在 1280px、1440px、1920px 均无重叠；
- 所有高级分析模块展示相同当前范围；
- 输入速度和时序回放明确使用当前范围结束日；
- 旧功能没有入口丢失或交互回归。

### 阶段 5：视觉收敛与回归

实施内容：

- 逐区对比 1440px 全页截图；
- 校准侧栏宽度、主内容边距、卡片高度、字号、行高和颜色；
- 检查加载时布局稳定性；
- 完善键盘焦点、ARIA 标签、禁用态和错误提示；
- 清理不再使用的旧 DOM 和 CSS，不删除后端路由；
- 更新相关可视化文档。

阶段验收：

- 页面信息层级、颜色、卡片密度和主要图表比例与参考图一致；
- 无控制台错误、未处理 Promise 或重复请求风暴；

## 9. 代码组织建议

本轮不改变目录层级，继续保持现有模块职责：

- `visualize/public/index.html`：页面语义结构、侧栏、模块容器和可访问性属性；
- `visualize/public/css/style.css`：设计令牌、桌面布局、组件、图表、主题和打印样式；
- `visualize/public/js/main.js`：全局范围、总览聚合、趋势图联动、导出和页面导航；
- `visualize/public/js/keyboard.js`：键盘布局模式、热力图、缩放、刷新和 PNG；
- `visualize/public/js/analysis.js`：高级分析、活跃概览、实时刷新、回放和范围对比；
- `visualize/public/js/calendar.js`：当前范围日历、单日选择和 Tooltip；
- `visualize/public/js/api_cache.js`：继续负责请求去重、短期缓存和主动失效。

相似日期解析、范围过滤、数字格式化等纯函数应收敛到一个明确位置，避免在 `main.js`、`calendar.js` 和 `analysis.js` 各自复制不同版本。若需要跨文件共享，优先增加一个小型原生工具脚本，不引入模块打包器。

## 10. API 复用方案

不调整后端接口，前端继续使用：

| API | 用途 |
| --- | --- |
| `/api/info` | 数据库可用范围和基础信息 |
| `/api/daily-stats` | 总览、日期趋势、活跃概览、每日统计 |
| `/api/keys` | 当前范围 Top 20 |
| `/api/heatmap` | 当前范围键盘热力图 |
| `/api/hourly-heatmap` | 星期 × 小时热力图 |
| `/api/region-stats` | 按键分区 |
| `/api/hand-stats` | 左右手统计 |
| `/api/combos` | 快捷键组合 |
| `/api/speed` | 当前范围结束日的输入速度 |
| `/api/timeline` | 当前范围结束日的回放数据 |

`/api/daily-stats` 返回全量每日数据，前端首次加载后缓存并按日期过滤。实时刷新或显式刷新时失效对应缓存，不为每张总览卡发起独立请求。

## 11. 边界与异常处理

- 空数据库：总览显示 0，日期控件禁用，各图表显示统一空状态；
- 单日数据：日期柱形可见，Brush 不强制要求两天宽度；
- 稀疏日期：补零日期参与自然日平均，但不计入活跃天数；
- 大数值：使用千分位，表格列不换行，Tooltip 保留完整值；
- 少于 20 个按键：排行按真实数据结束；
- API 失败：模块就近显示错误，保留其他成功模块，不使用空 `catch`；
- 请求竞争：对范围刷新增加版本号或取消标识，较早返回的数据不得覆盖较新的选择；
- 实时刷新：循环定时器内不记录重复日志，只记录启停、范围和异常；
- 主题存储失败：记录警告并继续使用当前主题，不阻断页面初始化；
- PNG/CSV/JSON 导出失败：显示可恢复错误并记录异常原因。

## 13. 风险控制

- 设计图数据存在示意性质，所有数值必须以本文口径和真实 API 数据为准；
- 旧功能保留会使最终页面比参考图更长，旧功能只能放在所属模块的扩展区域，不得挤占设计主网格；
- 日期范围是全页核心状态，禁止由各模块独立修补联动问题；
- 键盘布局和热力覆盖共享坐标，布局切换必须先完成坐标模型再调整视觉；
- 不为追求视觉效果引入远程字体、图标库、动画库或新的前端框架；
- 不修改归档目录 `archive/node-server/visualize/`；
- 如实现过程中发现必须修改后端接口或扩大数据模型，应暂停并单独确认。
